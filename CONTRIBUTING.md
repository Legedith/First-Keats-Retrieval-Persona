# Contributing

This project is a human-led life-narrative elicitation method. Contributions should improve safety, clarity, cultural range, or practical usability.

## Good contributions

Good contributions include:

- Safer question rewrites.
- New cultural or regional modules.
- Better consent language.
- Better templates.
- Interviewer training exercises.
- Evaluation rubrics.
- Research notes with citations.
- Examples of ordinary-life prompts.
- Accessibility improvements.
- Translation guidance.

## Contributions to avoid

Do not contribute:

- Prompts that pressure disclosure.
- Prompts that assume trauma.
- Prompts that imply one correct life path.
- Prompts that romanticize suffering.
- Prompts that treat all families as safe.
- Prompts that simulate the narrator.
- Voice-cloning workflows.
- First-person persona instructions.
- AI training workflows using narrator material without explicit consent.

## Prompt contribution rules

Every new prompt should be checked against this safety filter:

1. Does it insert a fact?
2. Does it insert an emotion?
3. Does it insert a motive?
4. Does it imply a desired answer?
5. Does it pressure disclosure?
6. Does it ask multiple questions?
7. Does it ignore culture, class, gender, caste, religion, or historical context?
8. Does it create privacy risk for living third parties?

If yes, rewrite.

## Adding research

When adding research notes:

- Prefer primary sources.
- Summarize what the method borrows.
- Do not overclaim.
- Distinguish evidence from interpretation.
- Add full links where possible.
- Note limitations.

## Cultural modules

A cultural module should not stereotype.

It should help interviewers ask about locally important domains such as:

- Kinship terms.
- Migration.
- Language.
- Caste or class.
- Religion.
- Gender norms.
- Work forms.
- Food.
- Festivals.
- Land.
- Household structure.
- Respectability.
- Colonial or political history.
- Community obligations.

Use open questions. Do not assume the narrator identifies with the category.

## Tone

Write plainly. Avoid academic fog. Avoid startup language. Avoid therapeutic promises.

The method should feel serious, respectful, and usable.
